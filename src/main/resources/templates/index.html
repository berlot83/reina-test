API SYNA

Documentación Reina API SNyA:

Todos los path menos este index están protegidos por Contraseña por medio de Spring security por capas, al sólo hecho de proteger los datos sensibles, luego muy posiblemente se quite la misma.

Datos:

  • Host ==> http://186.19.113.224:8080/
  • Path principal ==> http://186.19.113.224:8080/api/
  • Tener en cuenta que algunas respuestas pueden tardar ya que la DB es gigante.
  • A cada llamada que se realice a cualquiera de estos endpoints debe estar acompañada de un Token acoplado a cada request dentro del Header y No en el body, el mismo debe contener los siguientes atributos, Nombre, Apellidos, Email, Usuario, Rol, Sector.

    El atributo Rol sólo puede llevar los siguientes roles "temporarios" hasta que se definan por completo, ROL_FUNCIONARIO, ROL_CONSULTA, ROL_VISTA_LEGAJO, ROL_VISTA_LEGAJO_UBICACION, ROL_VISTA_LEGAJO_TECNICO.

    Si el rol es diferente devolverá Http Error 401, código de no autorizado
  • En la path de pruebas /asociador abrir la consola con F12 para ver los resultados específicos del JSON de resultados. En los cuadros sólo se muestran datos acotados a modo de pruebas y demostraciones.
